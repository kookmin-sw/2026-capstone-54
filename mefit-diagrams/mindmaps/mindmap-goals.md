## 목적

본 다이어그램은 캡스톤 프로젝트의 **세부 목표 23 개** 를 한눈에 시각화합니다. 사용자 (U) / 시스템 (S) / 기술 (T) / 운영 (O) 의 4 영역으로 분류하여 목표 간 종속 관계와 우선순위를 트리 형태로 표현합니다.

## 표현 내용

- **U (사용자) 5 목표** — 가입 10분 / 면접 15~20분 / 리포트 2분 / 주1회 보장 / 점수 *왜* 설명
- **S (시스템) 6 목표** — 24/7 무중단 / 수평 확장 / 장애 격리 / 이벤트 영상 처리 / 실시간 통신 / LLM 비용 추적
- **T (기술) 7 목표** — 마이크로서비스 / LLM 응용 / Property test / AWS 이벤트 / k3s on EC2 / CI/CD / 다중 LLM Provider
- **O (운영) 5 목표** — 1인 운영 / 시크릿 관리 / 즉시 롤백 / 1명령 부팅 / 공유 네트워크

## 활용 시점

- 보고서 §2.1 목표 명세 시
- 발표 자료 — *"무엇을 달성하려고 했는가"* 첫 슬라이드
- 진행 평가 — 23 목표 충족률 추적
